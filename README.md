# Ветки проекта

### Основные ветки:

- **`main`** — стабильная production-ветка. **Не коммитим напрямую!** До первого релиза не трогаем.
- **`dev`** — основная ветка для разработки. Сливаем свои фичи только после код-ревью и одобрения тимлида.
- **`release/*`** — релизные ветки. Создаёт тимлид по мере развития проекта.

### Рабочие ветки:
- **`server`** — база для серверной разработки 
- **`client`** — база для клиентской разработки

## Рабочий процесс c ветками

### Начало работы над задачей

1. **Выберите базовую ветку** в зависимости от типа задачи:
```bash
   # Для работы с бэкендом (Express, Sequelize):
   git checkout server
   git pull origin server
   
   # Для работы с фронтендом (React, Vite):
   git checkout client
   git pull origin client
```
2. **Создайте локальную ветку** от соответствующей базовой ветки:
```bash
git checkout -b name/краткое-описание-задачи
```

3. **Регулярно коммитьте свою работу**:
```bash
git add .
git commit -m "краткое-описание-работы"
```
4. **Регулярно обновляйте свою ветку с базовой**:
```bash
# Для серверных задач:
git pull origin server

# Для клиентских задач:
git pull origin client
```

## Завершение задачи
Пушим свою ветку в репозиторий:

```bash
git push origin name/ваша-ветка
```

Создаём Pull Request (Merge Request) из вашей ветки в соответствующую базовую ветку (server или client).
После одобрения тимлида ветка будет вмержена.

## ( * ) Полезное

**Часто используемые:**
``` bash
# Переключение на последнюю ветку
git checkout -

# Просмотр состояния файлов
git status

# Просмотр истории коммитов
git log --oneline -10

# Просмотр изменений
git diff
```

**Для работы с ветками:**
``` bash
# Список всех веток
git branch -a

# Удаление локальной ветки
git branch -d имя-ветки

# Удаление удаленной ветки
git push origin --delete имя-ветки
```

**( ! ) НЕ делайте прямое слияние** в:
- main
- dev
- server
- client


**НЕ пушите напрямую в базовые ветки:**

``` bash
# НЕПРАВИЛЬНО:
git push origin server

# ПРАВИЛЬНО:
git push origin feature/your-feature
НЕ забывайте обновлять свои ветки:
bash

# Перед началом работы всегда делайте:
git pull origin server  # или client
```

**Забыли откуда создали ветку?**
```bash
git log --oneline --graph -10
```

**Что-то пошло не так?**
```bash
# Отмена последних изменений в файлах
git checkout -- имя-файла

# Отмена индексации файлов
git reset
```


## ( ! ) Напоминания

- Коммитьте часто — маленькие коммиты лучше больших

- Пишите понятные сообщения коммитов

- Перед ПР убедитесь, что ваш код работает

- Тестируйте свои изменения

- Общайтесь — если есть сомнения, спросите тимлида
